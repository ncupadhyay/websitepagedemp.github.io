Introduction of JDBC


  • JDBC को Sun Microsystem ने Develop किया था.
  • JDBC का पूरा नाम java Databases connectivity हे, JDBC java की एक टेक्नोलॉजी हे,यह java J2SE (java 2 Stander Edition) के अंतर्गत आता हे.
  • JDBC का उपयोग करके हम java एप्लीकेशन को रिलेस्नल Databases के साथ connect कर सकते हे,और डेटाबेस से डेटा को access और डेटा store कर सकते हे।
  • JDBC का उपयोग करके किसी भी relational डेटाबेस से डेटा को access किया जा सकता हे।

What is JDBC ?


  • JDBC is a collection of API (application programmable interface), और ये API और कुछ नहीं classes होती हे।
  • JDBC जावा की एक टेक्नोलॉजी है, ओर इसी टेक्नोलॉजी के आधार पर कई सारे ORM (object Relational Mapping) Tool का देवेलोप्मेंट हुआ है
  • JDBC का उपयोग Java प्रोग्राम और डेटाबेस के बीच कनेक्टिविटी स्थापित करने के लिये किया जाता हे।
  • वर्तमान समय में कई सारे ORM tool जिनके द्वारा डेटाबेस पर ऑपरेशन किये जा सकते है पर ये सभी internally JDBC का उपयोग करते हे।

    • ** Example: Hibernate

Why we need JDBC?


अगर हम कोई भी प्रोग्राम या एप्लीकेशन को बनाते हे, तो हमें किसी न किसी डेटा को store करने के लिये डेटाबेस की आवस्य्क्ता होती चूँकि डेटाबेस मे डेटा स्थायी रुप से store रहता हे, इसलिये जावा प्रोग्राम से डेटा को डेटाबेस मे store करने के लिये JDBC की आवस्य्क्ता होति हे।

Who is responsible for implementing JDBC API ?


डेटाबेस vendor JDBC API को implement करने के लिये responsible होता हे।

Uses of JDBC


  1. Java Applets
  2. Servlet
  3. JSP (java server page)
  4. EJB (Enterprise Java Beans)
  5. Java Application

Advantages of JDBC


  • JDBC API प्लेटफॉर्म independent होती हे.
  • JDBC API डेटाबेस independent होती हे.
  • 3)JDBC का उपयोग करके किसी भी डेटाबेस पर ऑप्रेसन परर्फोर्मे किये जा सकते हे.
  • 4)JDBC का उपयोग करके CRUD ऑप्रेसन परर्फोर्मे किये जा सकते हे

Disadvantages of JDBC


  • JDBC मे लिखा गया प्रोग्राम database dependent होता हे.
  • JDBC में डेटाबेस को change करना मुस्किल होता है.
  • JDBC का code एक-दूसरे पर निर्भर होता है.
  • JDBC मे Exception handling अनिवार्य होती, बिना Exception handling क़े प्रोग्राम compile नही होता हे.
